## 🧠 How I think about quality

For me, QA is not just about finding bugs —  
it's about understanding the system, the users, and the risks.

I like to approach testing from multiple angles:
- 👤 User perspective — how real users might break the system
- ⚙️ Technical perspective — how things actually work underneath
- 🔥 Destructive mindset — what happens if everything goes wrong?
